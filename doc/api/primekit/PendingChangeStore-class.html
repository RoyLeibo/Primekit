PendingChangeStore

PendingChangeStore class final

Persists SyncChange objects to SharedPreferences so that unsynced local writes survive app restarts.

The store operates as a FIFO queue: changes are appended via enqueue and consumed (in order) via dequeueAll. After a successful remote push the caller should invoke clear to remove the flushed changes.

final store = PendingChangeStore();
await store.enqueue(change);
final pending = await store.dequeueAll();
// ... push to remote ...
await store.clear();
